1. Platform Vizyonu

Operations Platform, yalnızca hata izleme aracı değildir. Entegrasyon operasyonlarının merkezi yönetim katmanı olmayı hedefler.

“Know what failed, why it failed, who is affected, and what to do next.”

Platform; ham event, API response, webhook, deployment, application log, support ticket ve email alarmını aynı olay zincirinde birleştirir. Daha sonra incident oluşturur, ilgili kanıtları toplar, AI ile açıklama üretir ve ekiplerin çözüm sürecini yönetmesini sağlar.

2. Çözülen Problem

Bir entegrasyon bozulduğunda ekipler genellikle farklı sistemlere dağılmış kanıtları manuel toplar:

  • API response ve status code
  • Webhook delivery logları
  • Son deployment zamanı
  • Credential veya config değişikliği
  • Application logları
  • Benzer geçmiş incident
  • Müşteri ticket'ları
Mevcut araçlar çoğunlukla “hata oldu” der. Bu ürün “hata neden olmuş olabilir ve hangi kanıtlar bunu destekliyor?” sorusuna cevap verir.

4. Sisteme Giren ve Sistemden Çıkan Veri

Girdiler

  • Integration event
  • HTTP status code
  • Request/response metadata
  • Latency
  • Deployment event
  • Config/secret change metadata
  • Application log

Çıktılar

  • Incident category
  • Severity
  • Affected integration
  • Affected request/customer count
  • Probable root cause
  • Evidence list
  • Recommended actions
  • Confidence + human review flag

Örnek AI Çıktısı

{
  "incidentTitle": "Stripe authentication failures after deployment",
  "category": "AUTHENTICATION_ERROR",
  "severity": "HIGH",
  "affectedIntegration": "Stripe Payments",
  "affectedRequests": 83,
  "probableRootCause": "Production API secret changed during the latest deployment.",
  "evidence": [
    "First 401 occurred 2 minutes after deployment",
    "No 401 errors in previous 24 hours",
    "Secret version changed in deployment metadata"
  ],
  "recommendedActions": [
    "Verify the current production secret",
    "Compare it with the previous secret version",
    "Rollback configuration if mismatch is confirmed"
  ],
  "confidence": 0.92,
  "needsHumanReview": true
}

11. Claude ile Geliştirme Planı

Claude'a bütün ürünü tek seferde yazdırma. Her milestone ayrı ilerlemeli; her aşamada test, açıklama ve code review istenmeli.

Claude'a Verilecek Ana Bağlam

You are my senior .NET pair programmer and software architect.

We are building:
AI Integration Failure Intelligence

Core problem:
When a SaaS integration fails, the product should correlate technical evidence and explain what likely broke, why it happened, who is affected, and what to do next.

Tech stack:
- ASP.NET Core
- C#
- EF Core
- PostgreSQL
- Redis
- Hangfire
- Semantic Kernel / OpenAI
- Serilog
- OpenTelemetry
- Docker Compose

Architecture rules:
- Start as a modular monolith, not microservices.
- Use Clean Architecture pragmatically.
- AI must use structured outputs.
- AI may only reason over supplied evidence.
- No automatic remediation in the MVP.
- Human review is required.
- Every milestone must include tests.
- Do not generate the whole product at once.

First task:
Create the solution structure, domain entities, initial database model, Docker Compose, and a short architecture decision record. Do not implement AI yet.

M6Serilog, Seq, OpenTelemetry, health checks, demo scenario
